Insights

Sales Over Time

When looking at monthly sales in 2017, there are comparatively minimal sales in the beginning of the year with a steady increase as the year progresses. There is a sharp increase in sales in October and November, with a small decrease in December. This increase in sales during this period aligns with the holiday season.

Best Selling Categories

Across all reported Olist sales in 2017, Home Good products were the top selling item at R$1,489,194. Electronics and Health & Beauty products followed closely behind at R$1,220,554 and R$1,219,905 respectively.

Top States by Sales

Of all the sales reported in this data set, the vast majority of sales were made in São Paulo at R$6,545,602. The second top selling state, Paraná, had a total of R$978,319. São Paulo also contained the largest number of total orders, sellers, and number of different products sold.

Overview

Per the description provided with the Kaggle data set Olist is “the largest department store in Brazilian marketplaces. Olist connects small businesses from all over Brazil to channels without hassle and with a single contract.” In this project, I analyzed the impact various factors such as time of year, region, and category had on sales.

Data Cleaning

Remove Unnecessary Characters

The customer, geolocation, order items, order payments, orders, products, product translation, and sellers tables all contained quotation marks and underscore symbols in multiple different columns. To remove them, I used the replace function with empty quotation marks as shown below.

#########clean data for customer table#########
SELECT * FROM olist_data.customer;
											
#get rid of quotation marks around customer id columns and zip code
UPDATE olist_data.customer
SET customer_id = REPLACE(customer_id, '"', '') 
WHERE customer_id LIKE '"%"';
								
UPDATE olist_data.customer
SET customer_unique_id = REPLACE(customer_unique_id, '"', '') 
WHERE customer_unique_id LIKE '"%"';
											
UPDATE olist_data.customer
SET customer_zip_code = REPLACE(customer_zip_code, '"', '') 
WHERE customer_zip_code LIKE '"%"';

Multiple Order IDs

When looking at the data, I found that the unique identifier, order ID, was duplicated throughout the Order Items table. These rows with duplicate order IDs contained unique information in all the other columns. I then confirmed the Orders table and Order Payment table did not have duplicate order IDs. After looking through randomly selected order IDs that were duplicated in the Order Items table, I found that these multiple order IDs indicated a single order made by a customer containing multiple different items.

select order_id, count(*)
from olist_data.order_items
group by order_id
having count(*) > 1;
#3236 rows returned
												
select order_id, count(*)
from olist_data.orders
group by order_id
having count(*) > 1;
#No rows returned
												
select order_id, count(*)
from olist_data.order_payments
group by order_id
having count(*) > 1;
#No rows returned

Checking Payment Value

To make sure that the freight value and price matched the reported payment value for the orders with multiple items, I utilized the calculation below to make sure that the values were the same.

#########Make sure total value and payment value match#########
select order_id, count(*), sum(total_value)*count(*), sum(payment_value)
from calculated_total_value
where total_value != payment_value
group by order_id
having count(order_id) > 1;

Creating Calculated Table

In order to get a complete total cost for a given order, I needed to add the freight value and the price of the product. Additionally, I wanted to have all the relevant product information in one table. To accomplish this, I created a new view..

create view complete_calculated_total_value as
SELECT order_id, order_item_id, calculated_total_value.product_id, seller_id, shipping_limit_date, price, 
freight_value, total_value, payment_value, broad_category, product_category_name_english, product_category_name
FROM olist_data.calculated_total_value
LEFT JOIN product_translations
ON calculated_total_value.product_id = product_translations.product_id;

Add State Names

The original data set only provided abbreviations for the state names. In order to show state names on the final dashboard, I edited the geolocation CSV in Excel. I created a new State column and added the corresponding state names alongside the abbreviations.

Dashboard Building

Incomplete Data for 2016 & 2018

Although the original data set included data from 2016 - 2018, I decided to only include sales data for 2017. This is because, when first visualizing sales and orders over time, I noticed there was a significant decrease in sales from September to December compared to the rest of the year. When looking at each year individually, I found that for 2016, there were some sales in September, October, and December. For 2018, sales data was given for January to September, with a sharp decline in September sales. Only 2017 had a complete year's sales reported. Therefore, I only included 2017 in the final dashboard to clearly indicate an accurate representation of sales for Olist over the course of a year.

Best Sellers Categories

In order to have the correct name appear for Best Selling Category and Best Selling State, I create a calculated field named Rank by Sales: Rank(SUM([Total Value])). I then applied the calculation as a filter and set it to only show one value. This allowed for only the name of the Best Selling Category and Best Selling State to be shown on the dashboard.
